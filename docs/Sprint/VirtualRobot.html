Introduction

Goal: Realizzare un sistema software che comanda un Differential Drive Robot (DDR).

Requirements

R1   Partendo dalla posizione iniziale HOME, il robot si sposta lungo il perimetro di una stanza rettangolare vuota.

R2   Il DDR si deve fermare quando tornato in HOME.

R3   Il DDR è anche sensibile ai dati rilevati da un Radar. Quando il Radar rileva un 'intruso' a una distanza minore di una distanza prefissata ``DMIN``:
- se il DDR si sta muovendo , il DDR si ferma fino a che il Radar non rileva più un 'intruso' così vicino.
- se il DDR è fermo in HOME, il DDR ruota su sè stesso fino a che un 'intruso' così vicino scompare.

Requirement Analysis

Nel sistema entrano in gioco due principali entità:

- Il Robot (DDR), una macchina (già fornita) in grado di spostarsi in avanti e indietro, di ruotare 
  e di fermarsi in modo autonomo.
  Inizia a muoversi da una posizione iniziale HOME, si sposta lungo un percorso
  e si ferma quando torna al punto di partenza.
  Esso, inoltre, deve essere in grado di fermarsi o ruotare (a seconda del caso)
  se si imbatte in ostacoli abbastanza vicini.

- Per formalizzare la HOME, dobbiamo prima formalizzare il concetto di stanza. La
  stanza ha una forma rettangolare, pertanto possiamo rappresentarla come una griglia.
  Per semplicità, scegliamo di rappresentare come unità di misura della stanza l'unità
  robot. Il robot può essere rappresentato inscrivibile in un quadrato di lato r. Ogni
  cella (piastrella) della griglia (stanza) avrà quindi misura r*r.

  - Il Radar, un dispositivo dotato di sensori in grado di captare oggetti (ostacoli
  in questo caso) posti a una certa distanza.
  Se l'ostacolo si trova a una distanza minore di "DMIN", il compito del Radar è
  quello di notificare l'ostacolo al Robot stesso.
  Può essere visto come un attore: ha una vita propria e fa le cose proprie in autonomia.
  Come fa il radar a comunicare con il DDR? Attraverso tre modi possibili:
    - Evento (Emissione di informazioni) (Modello A): il radar notifica il Robot attraverso
      un evento ogni tot tempo.
    - Request/Response (Polling) (Modello B): il Robot fa una richiesta e attende che il radar
      risponda.
    - Observer (publish/subscrive) (Modello C): il Robot si iscrive e il Radar notifica
      automaticamente le informazioni (pubblica).

Radar con evento

    Event dist : dist(X)   

    QActor radarevent context ctxradar { 
        
        State s0 initial { 	
            //stato iniziale
            //si inizializza timer (ogni quanto manda la distanza)
        } 
        Transition t0 
            whenTimeVar  Timer -> sendDist
        
        State sendDist{
                //controlla se c'è u  oggetto davanti e se si:
                //notifica a robot
                emit dist : dist(X)
        }
        Goto s0
    }

Radar con polling

    Request getDistance : getDistance(X)
    Reply dist : dist(X) for getDistance

    QActor radarpolling context ctxradar { 

	State s0 initial {
        //stato iniziale
 		println("Aspetto richiesta...") color green
	}
	Transition t0  
        //arriva richiesta -> risponde con la distanza
		whenRequest getDistance -> handleRequest
   
	State handleRequest{  
		printCurrentMessage	color green	 
 		onMsg( getDistance : getDistance(X) ){  
 			// calcola distanza 
 			// ...
 			//manda risposta
 			replyTo getDistance with dist : dist( $Answer )  
 		} 
	}  
	Goto s0
}

Radar con observer

    Dispatch dist : dist(X)

    QActor radarobserver context ctxradar{
        State s0 initial { 	
            //stato iniziale
            //si inizializza timer (ogni quanto pubblica la distanza)
        } 
        Transition t0 
            whenTimeVar  Timer -> publishDist
            
        State publishDist {
            //aggiorna distanza
            updateResource [# "$DIST " #]
        }
        Goto s0
    }
